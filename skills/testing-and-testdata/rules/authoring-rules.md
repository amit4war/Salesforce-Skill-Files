# Authoring Rules

- Write tests as behavior specs, not coverage fillers.
- Keep test method names scenario-specific and readable.
- Use minimal data needed for each scenario.
- Assert both return values and database side effects.
- Always include failure-path assertions where applicable.

## Output Format

1. Scenario matrix
2. Data setup approach
3. Test methods added/updated
4. Assertion highlights
5. Coverage/regression notes
