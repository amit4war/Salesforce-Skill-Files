# Authoring Rules

- Use action-oriented steps, not generic advice.
- Always state chosen data pattern (`wire`, LDS, Apex) and why.
- Define all UI states explicitly.
- Document event contracts and payload shape.
- Specify expected outputs and test evidence.

## Output Format

1. Component scope and UX goal
2. Data strategy
3. UI state behavior
4. Files changed
5. Tests and validation
6. Risks and next checks
