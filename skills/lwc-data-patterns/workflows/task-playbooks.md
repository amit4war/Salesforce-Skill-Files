# Task Playbooks

## Playbook A: Build New Data-Driven LWC

1. Define view and actions.
2. Choose data access strategy.
3. Implement reusable UI skeleton.
4. Add state handling and tests.

## Playbook B: Refactor Existing LWC

1. Identify duplicated logic and weak states.
2. Move reusable parts into shared modules/components.
3. Standardize events and navigation.
4. Add regression tests.

## Playbook C: Add Apex-Backed Action

1. Define input/output contract.
2. Implement call and error mapping.
3. Enforce security in Apex.
4. Validate with Jest + Apex tests.
