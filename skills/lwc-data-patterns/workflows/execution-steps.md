# Execution Steps

1. Capture component purpose, user actions, and target surface.
2. Select data strategy (`wire`, LDS, Apex) based on complexity.
3. Define state model: loading/empty/success/error.
4. Confirm reusable component opportunities.
5. Implement template, controller, and metadata config.
6. Add event and navigation behavior.
7. Add Jest tests and Apex tests (if Apex used).
8. Validate accessibility and responsiveness.
