# Authoring Rules

- Use imperative execution steps with command-first guidance.
- Always include decision rationale for risk and test level.
- Separate validation commands from deploy commands.
- Include expected outputs for each step (job ID, test summary, status).
- Include rollback and post-deploy verification in every final response.

## Output Format

1. Scope and target org
2. Risk classification
3. Commands executed/planned
4. Validation outcomes
5. Deploy outcomes
6. Rollback and next checks
